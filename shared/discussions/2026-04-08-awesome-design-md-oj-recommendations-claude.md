## 详细推荐

### 1. Linear — 全站整体视觉框架

**仓库路径**: `design-md/linear.app/DESIGN.md`

**为什么适合 OJ 项目**

Linear 是专为工程师打造的极致暗色设计系统。OJ 的核心用户（学生和教师）本质上是开发者群体，Linear 的暗色原生美学与这一用户画像高度契合。

关键设计令牌对 OJ 的价值：

- **近黑色背景 `#08090a`** — 长时间刷题、盯排名的场景下，暗色主题显著降低视觉疲劳
- **Inter Variable + OpenType `"cv01", "ss03"`** — 高可读性无衬线字体，适合代码描述和题目文本
- **Berkeley Mono 等宽字体** — 代码展示、判题输出的天然伴侣
- **半透明白色边框 `rgba(255,255,255,0.05~0.08)`** — 暗色面板间用极细边框分隔，适合题目列表、提交列表等密集数据展示
- **品牌靛紫 `#5e6ad2` / `#7170ff`** — OJ 可借鉴为 Accept（通过）的强调色
- **五级文字亮度阶梯** (`#f7f8f8` → `#d0d6e0` → `#8a8f98` → `#62666d`) — 完美映射 OJ 场景：题目标题、题目描述、时间限制、题号标签
- **药丸徽章 (9999px radius)** — 天然适合题目难度标签（Easy/Medium/Hard）、AC 率百分比、语言标签

**适用 OJ 模块**: 全站整体框架、导航栏、侧边栏、题目列表页、提交列表页

**需要注意**: Linear 极度克制（几乎无彩色），如果 OJ 需要更丰富的视觉层次（如不同难度的颜色编码），需要在此基础上扩展。

### 2. Vercel — 排行榜与数据表格

**仓库路径**: `design-md/vercel/DESIGN.md`

**为什么适合 OJ 项目**

Vercel 的设计系统以精密著称，其多层阴影栈（shadow-as-border）技术特别适合需要展示结构化数据的排行榜页面。

关键设计令牌对 OJ 的价值：

- **影子即边框 `box-shadow: 0px 0px 0px 1px rgba(0,0,0,0.08)`** — 排行榜行项之间的分隔线可使用此技术，视觉更精致
- **Geist 字体族 + 极致负字距 (-2.4px at 48px)** — 排名数字的大号展示用压缩字距，视觉冲击力强
- **三色工作流颜色** — Ship Red (`#ff5b4f`)、Preview Pink (`#de1d8d`)、Develop Blue (`#0a72ef`) 可映射为 OJ 的三种判题状态：Wrong Answer、Pending/Running、Accepted
- **指标卡片组件** — 大号数字 + 描述文字的卡片模式，天然适合「解题数」「AC 率」「排名」等个人统计
- **白色背景 + 极简排版** — 适合排行榜这类需要高信息密度、快速扫读的页面

**适用 OJ 模块**: 排行榜、比赛积分榜、用户统计面板、首页数据展示

**与 Linear 的搭配**: 排行榜和公共页面使用 Vercel 的白色主题（亮色模式），代码编辑器和后台使用 Linear 的暗色主题，两者切换形成明暗对比。

### 3. Supabase — 代码编辑器与提交详情

**仓库路径**: `design-md/supabase/DESIGN.md`

**为什么适合 OJ 项目**

Supabase 是以开发者为中心的暗色平台，其祖母绿强调色系统与代码/终端美学高度吻合 OJ 的代码提交场景。

关键设计令牌对 OJ 的价值：

- **祖母绿品牌色 `#3ecf8e` / `#00c573`** — 天然映射 OJ 的「Accepted / 通过」状态，绿色在判题系统中是正反馈的核心颜色
- **Source Code Pro 大写技术标签** — 适合判题结果中的 TIME_LIMIT、MEMORY_LIMIT、RUNTIME_ERROR 等技术标签
- **HSL 色彩令牌系统 + Alpha 通道** — 半透明绿色边框 `rgba(62, 207, 142, 0.3)` 可用于高亮 AC 的题目行
- **Circular 字体的极致 Hero 行高 1.00** — 适合比赛倒计时、大号排名数字等需要视觉冲击力的场景
- **药丸主按钮 (9999px)** — 「提交代码」按钮可用此样式
- **边框层级深度系统** (`#242424` → `#2e2e2e` → `#363636`) — 不用阴影的深度表达，适合代码面板和判题结果的多层嵌套

**适用 OJ 模块**: 代码编辑器面板、提交详情页、判题结果展示、题目通过状态标记

### 4. Sentry — 提交状态面板与判题仪表盘

**仓库路径**: `design-md/sentry/DESIGN.md`

**为什么适合 OJ 项目**

Sentry 是错误追踪平台，其数据密集型暗色仪表盘设计与 OJ 的判题结果分析场景高度相似。

关键设计令牌对 OJ 的价值：

- **深紫黑背景 `#1f1633`** — 比 Linear 的纯暗色更温暖，适合长时间使用的判题面板
- **柠檬绿强调色 `#c2ef4e`** — 可用于高亮 AC（Accepted）状态，与紫色背景形成高对比
- **珊瑚/粉色强调色 `#ffb287` / `#fa7faa`** — 可映射为 WA（Wrong Answer）和 RE（Runtime Error）状态
- **玻璃态效果 `blur(18px) saturate(180%)`** — 适合浮动的判题详情面板
- **内凹阴影按钮** — 提交按钮的按压感，提供交互反馈
- **Monaco 等宽字体** — 判题输出和编译错误信息展示
- **大写字母 + 字间距 0.2px 的系统化标签** — 适合判题状态标签如 ACCEPTED、WRONG_ANSWER、TIME_LIMIT_EXCEEDED

**适用 OJ 模块**: 提交状态实时面板、判题结果详情页、错误追踪面板、管理后台判题监控

**注意**: Sentry 的紫色调偏重，建议仅用于判题相关的数据密集面板，而非全站主视觉。

### 5. Notion — 题目展示与讨论区

**仓库路径**: `design-md/notion/DESIGN.md`

**为什么适合 OJ 项目**

OJ 的题目页本质上是长文本阅读体验——题目描述、输入输出说明、样例数据、提示信息。Notion 的温暖极简排版系统是这种场景的最佳参考。

关键设计令牌对 OJ 的价值：

- **温暖中性色阶** (`#f6f5f4` 暖白、`#31302e` 暖黑、`#615d59` 暖灰) — 长时间阅读题目时眼睛更舒适
- **NotionInter 字体的负字距压缩** — 题目标题可用 48px/700 weight 的压缩排版
- **超薄边框 `1px solid rgba(0,0,0,0.1)`** — 题目描述区与代码区之间的分隔
- **多层阴影栈（单层不透明度 ≤ 0.05）** — 题目卡片和讨论帖卡片的柔和浮动感
- **暖白/纯白交替背景** — 题目描述区（暖白）与代码区（纯白）的视觉分隔
- **Notion Blue `#0075de`** — 题目中的超链接颜色（如参考链接、外链题目）
- **药丸状态标签** — 难度标签（Easy/Medium/Hard）、算法分类标签（DP/Graph/Tree）
- **四层级字重系统** (400/500/600/700) — 题目中的层级清晰：题号 > 标题 > 小节标题 > 正文 > 样例

**适用 OJ 模块**: 题目展示页、题解编辑器、讨论区帖子、博客/Wiki、帮助文档

### 6. Stripe — 排行榜与竞赛统计

**仓库路径**: `design-md/stripe/DESIGN.md`

**为什么适合 OJ 项目**

Stripe 展示金融数据的设计能力在排行榜和竞赛统计场景中非常有价值——排名本身就是一种需要精密展示的「金融数据」。

关键设计令牌对 OJ 的价值：

- **自定义 sohne-var 字体 + `"ss01"` 造型集** — 排行榜标题和排名数字的独特排版
- **权重 300（超轻）的大标题** — 排行榜页头可用轻量级大字标题，优雅不笨重
- **蓝色调多层阴影 `rgba(50,50,93,0.25)`** — 排行榜卡片的精密深度感
- **深海蓝 `#061b31` 文字色** — 排名数字使用此色，比纯黑更有质感
- **Tabular 数字 `"tnum"`** — 排名、AC 数、用时等数字列对齐，表格数据必备
- **Success Green `#15be53`** — AC 状态标记
- **4-8px 保守圆角** — 排行榜表格行项不需要大圆角，保守圆角更正式
- **深品牌区 `#1c1e54`** — 竞赛页面头部可用此深色品牌背景

**适用 OJ 模块**: 排行榜页、竞赛统计面板、管理后台数据报表、评分详情

### 7. ClickHouse — 竞赛实况大屏与性能指标

**仓库路径**: `design-md/clickhouse/DESIGN.md`

**为什么适合 OJ 项目**

ClickHouse 的极端高对比度设计（霓虹黄绿 `#faff69` on 纯黑 `#000000`）特别适合竞赛排行榜的实时大屏展示和需要「视觉冲击力」的场景。

关键设计令牌对 OJ 的价值：

- **霓虹黄绿 `#faff69` on 纯黑** — 竞赛实时排行榜中，领先者的行高亮可用此极端对比
- **Inter 900 (Black) + 96px 超大标题** — 竞赛名称、大屏倒计时等需要「重磅」视觉的场景
- **森林绿 `#166534` CTA 按钮** — 「参加比赛」「提交代码」等行动按钮
- **性能指标大号数字展示** — 竞赛中的「已提交 X 次」「AC 率 Y%」「剩余 Z 分钟」等实时统计
- **Inset 阴影（按压感）** — 排行榜中当前用户行的凹陷效果
- **大写宽字距标签 1.4px** — 竞赛状态标签如 UPCOMING / RUNNING / ENDED

**适用 OJ 模块**: 竞赛实时排行榜大屏、竞赛首页 Hero、比赛倒计时、年度/月度统计

**注意**: ClickHouse 风格极端，建议仅用于竞赛大屏和需要视觉冲击力的特定场景，不适合全站。

### 8. PostHog — 用户主页与个人统计

**仓库路径**: `design-md/posthog/DESIGN.md`

**为什么适合 OJ 项目**

PostHog 的温暖、反传统的分析仪表盘美学，非常适合 OJ 的用户个人主页——它展示的不是冷冰冰的数据，而是有温度的学习旅程。

关键设计令牌对 OJ 的价值：

- **暖色鼠尾草/橄榄绿主色调** — 用户主页不想像后台那样冷硬，温暖色调传达「学习是友好的」
- **IBM Plex Sans Variable** — 技术可信度 + 人文温度的结合
- **隐藏品牌橙色 `#F54E00` 仅在 hover 出现** — 解题进度条 hover 时显示详细信息，交互惊喜
- **暖羊皮纸背景 `#fdfdf8`** — 用户主页的温暖底色
- **手绘风格插图元素** — OJ 成就徽章、解题里程碑可用类似风格
- **内容密集型编辑布局** — 用户的解题历史、提交统计、比赛参与记录等数据展示
- **超大 body 行高 (1.50–1.71)** — 个人简介、解题笔记等长文本阅读

**适用 OJ 模块**: 用户个人主页、个人统计面板、成就系统、学习进度页

### 9. Cursor — 代码视图与题目详情

**仓库路径**: `design-md/cursor/DESIGN.md`

**为什么适合 OJ 项目**

Cursor 的三字体系统（哥特体展示 + 衬线正文 + 等宽代码）精确映射了 OJ 题目页面的三重内容类型：题目标题、题目描述、代码/样例。

- **Hover 变红 `#cf2d56`** — WA 状态的视觉反馈

**适用 OJ 模块**: 题目详情页（展示+正文+代码三区）、题解页面、代码对比视图

## 实施建议

### 分层策略

推荐采用「基础层 + 场景层」的分层设计策略：

1. **基础层**: 采用 Linear 的暗色视觉框架作为全站基础（背景、导航、侧边栏、通用组件）
2. **场景层**: 不同模块叠加对应的设计系统特色
   - 题目区引入 Notion 的温暖排版令牌
   - 代码区使用 Supabase 的终端美学
   - 排行榜使用 Vercel/Stripe 的精密数据展示
   - 竞赛大屏使用 ClickHouse 的高对比度冲击力
   - 用户主页使用 PostHog 的温暖面板风格

### 颜色统一化

从各推荐系统中提取统一的状态颜色映射：

| 状态 | 颜色 | 来源 | 用途 |
|------|------|------|------|
| Accepted | `#3ecf8e` (Supabase Green) | Supabase | AC 状态、通过标记 |
| Wrong Answer | `#cf2d56` (Cursor Error) | Cursor | WA 状态、错误标记 |
| Pending/Running | `#0a72ef` (Vercel Develop Blue) | Vercel | 等待中、运行中 |
| Time Limit | `#ffb287` (Sentry Coral) | Sentry | TLE 状态 |
| Runtime Error | `#fa7faa` (Sentry Pink) | Sentry | RE 状态 |
| 题目难度 Easy | `#10b981` (Linear Emerald) | Linear | 简单题标签 |
| 题目难度 Medium | `#F7A501` (PostHog Gold) | PostHog | 中等题标签 |
| 题目难度 Hard | `#ff5b4f` (Vercel Ship Red) | Vercel | 困难题标签 |

### 后续步骤

1. 将本文档提交 Codex 评审
2. 确定最终采用的设计系统组合
3. 将选定 DESIGN.md 文件下载到项目 `references/design-md/` 目录
4. 基于选定系统创建 OJ 项目的统一 `DESIGN.md`，融合各系统中最适合 OJ 的令牌
5. 在前端重构的 P5（竞赛/排行榜）和 P2（题目管理）阶段中应用
