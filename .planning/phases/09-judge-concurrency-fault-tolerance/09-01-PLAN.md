       - `BreakerState` enum: `Closed`, `Open`, `HalfOpen`
       - `CircuitBreaker` struct with fields:
         - `failure_count: AtomicUsize`
         - `is_open: AtomicBool` (false = Closed/HalfOpen, true = Open)
         - `last_failure_time: std::sync::Mutex<Option<Instant>>`
         - `failure_threshold: usize` (default 5, per D-06)
         - `half_open_timeout_secs: u64` (default 30, per D-06)
       - `pub fn new(failure_threshold: usize, half_open_timeout_secs: u64) -> Self`
       - `pub fn allow_request(&self) -> bool`:
         - If `is_open` is false (Closed): return true
         - If `is_open` is true (Open): check elapsed time since `last_failure_time`. If elapsed >= `half_open_timeout_secs`, set `is_open` to false (transition to HalfOpen) and return true. Otherwise return false.
       - `pub fn record_success(&self)`: Set `failure_count` to 0, set `is_open` to false (close breaker)
       - `pub fn record_failure(&self)`: Increment `failure_count` by 1. If count >= `failure_threshold`, set `is_open` to true and update `last_failure_time` to `Instant::now()`.
       - `pub fn state(&self) -> BreakerState`: Read current state for monitoring/heartbeat reporting
       - `pub fn failure_count(&self) -> usize`: Read current failure count for monitoring
    2. Add `pub mod circuit_breaker;` to `judge-worker/src/main.rs` (after existing mod declarations)
    3. Write unit tests in the same file using `#[cfg(test)] mod tests` block. Tests use short timeouts (1 second) for half-open verification with `tokio::time::sleep`.
    4. Per D-05: No external state, no Redis dependency. Per D-07: Fail-open semantics are handled by the caller (not the breaker itself).

       pub async fn consume_priority(
           conn: &mut MultiplexedConnection,
           contest_stream: &str,
           normal_stream: &str,
           group_name: &str,
           consumer_name: &str,
       ) -> Result<Vec<(String, SubmissionMessage, String)>> {
           // Try contest stream first (non-blocking, per D-03)
           let contest_msgs = consume_submission(
               conn, contest_stream, group_name, consumer_name, None,
           ).await?;
           if !contest_msgs.is_empty() {
               return Ok(contest_msgs.into_iter()
                   .map(|(id, msg)| (id, msg, contest_stream.to_string()))
                   .collect());
           }
           // Fall back to normal stream (5s block, same as current)
           let normal_msgs = consume_submission(
               conn, normal_stream, group_name, consumer_name, Some(5000),
           ).await?;
           Ok(normal_msgs.into_iter()
               .map(|(id, msg)| (id, msg, normal_stream.to_string()))
