<objective>
Fix three critical/high gaps in the judge monitoring and DLQ system: DLQ retry type mismatch (CR-01), missing admin RBAC on judge monitor routes (Codex Critical), and DLQ metadata not threaded through processing loop (WR-05).

Purpose: The DLQ retry feature is currently non-functional — every retried entry fails to parse on the worker, creating an infinite retry loop. The admin judge monitor endpoints have no RBAC, allowing any authenticated user to view/retry/delete DLQ entries. DLQ entries lose origin metadata, breaking retry routing.

Output: Functional DLQ retry, admin-only judge monitor endpoints, correct DLQ metadata.
</objective>

Keep all existing fields (result_json, submission_id, etc.) as-is. The DLQ needs both: result_json for admin inspection of what went wrong, and original_message for re-enqueueing.

All four judge monitor route handlers (get_judge_status, list_dlq, retry_dlq, delete_dlq) currently accept only `State(state): State<AppState>`. They need admin RBAC.

While the routes are nested under the protected_router which has auth+tenant middleware (ensuring JWT is valid and Claims are in extensions), there is no role check. Any authenticated user can access these admin endpoints.

Add AuthExtractor + ensure_admin to every handler:

1. Add imports at the top of routes.rs:
```rust
use crate::middleware::auth::AuthExtractor;
use shared::models::Claims;
```

2. Add ensure_admin helper function (same pattern as users/routes.rs):
```rust
fn ensure_admin(role: &str) -> Result<(), AppError> {
    if role != "admin" && role != "root" {
        return Err(AppError::Auth("Admin access required".into()));
    }
    Ok(())
}
```

3. Add `AuthExtractor(claims): AuthExtractor` parameter to each handler, after State and before Path/Query. Call `ensure_admin(&claims.role)?;` as the first line of each handler body.

Example for get_judge_status:
```rust
async fn get_judge_status(
    State(state): State<AppState>,
    AuthExtractor(claims): AuthExtractor,
) -> Result<impl IntoResponse, AppError> {
    ensure_admin(&claims.role)?;
    // ... rest unchanged
```

Apply the same pattern to list_dlq, retry_dlq, and delete_dlq.
