<objective>
Create the standalone `backend/feature-gateway/` crate as an independent binary service.

Moves all feature gateway logic from `api-infra/src/feature_gateway/` into a new workspace crate following the `judge-worker` standalone binary pattern (D-17). The Gateway connects directly to PostgreSQL (D-19), serves HTTP on port 3001 (D-18), and authenticates API calls with WORKER_SECRET (D-24). Hosts all CRUD + resolve + health endpoints (D-25).

This plan does NOT modify the API server — coexistence is handled in Plan 02.

- .planning/phases/11-feature-gateway-infrastructure/11-CONTEXT.md — D-17 through D-26 decisions
</read_first>

<action>

1. Implement Axum middleware function `require_worker_secret()` that:
   - Reads `WORKER_SECRET` env var
   - Extracts `Authorization: Bearer <token>` from request headers
   - Returns 401 if missing or mismatched
   - Calls `next.run(req)` if valid

   ```rust
   pub fn require_worker_secret() -> impl Fn(Request, Next) -> Pin<Box<dyn Future<Output = Result<Response, StatusCode>> + Send>> + Clone
   ```

3. Add unit test: valid token passes, invalid token returns 401, missing header returns 401
</action>

<task id="5">
<title>Create Gateway main.rs binary entry point</title>
<action>
Create `backend/feature-gateway/src/main.rs` following judge-worker pattern:

1. Initialize tracing with EnvFilter
2. Read env vars: DATABASE_URL, FEATURE_GATEWAY_ENABLED (default "true"), GATEWAY_BIND_ADDRESS (default "0.0.0.0:3001"), WORKER_SECRET
3. Create PgPool with `PgPoolOptions::new().max_connections(5).connect_lazy(&database_url)`
4. Create `AppState { gateway: Arc::new(FeatureGatewayService::new(pool)) }`
5. Build Axum Router with `gateway_router(state)`
6. Bind to GATEWAY_BIND_ADDRESS, log "Feature gateway listening on {addr}"
7. Run with `axum::serve(listener, app).await`
</action>
<acceptance_criteria>
- `backend/feature-gateway/src/main.rs` contains `#[tokio::main]`
- Reads DATABASE_URL, FEATURE_GATEWAY_ENABLED, GATEWAY_BIND_ADDRESS, WORKER_SECRET from env
- Creates PgPool and FeatureGatewayService
- Starts Axum server on configurable port
- `cargo build -p feature-gateway` produces binary `target/debug/feature-gateway`
</acceptance_criteria>
</task>
