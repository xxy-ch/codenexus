    **1. Create `frontend/src/services/featureGateway.ts`**

    Define types:
    ```typescript
    export interface FeatureState {
      slug: string
      enabled: boolean
      source: 'default' | 'global' | 'campus' | 'grade' | 'class' | 'system_emergency_off'
    }

    export interface FeatureRegistryEntry {
      id: string
      slug: string
      name: string
      description: string | null
      default_enabled: boolean
      category: string
      created_at: string
    }
    ```

    Implement `useFeatureEnabled(slug: string)` hook:
    - Uses `useQuery` with queryKey `['features', 'resolved']` (shared cache -- one query for all features).
    - queryFn: `api.get('/features/resolved')` -> response.data (typed as `Record<string, FeatureState>`).
    - staleTime: 60_000 (1 minute).
    - retry: 1.
    - Returns `{ enabled: data?.[slug]?.enabled ?? true, isLoading }`. The `?? true` implements fail-open: defaults to enabled when data is undefined (loading or error). Per D-13.

    Implement `useFeatureRegistry()` hook:
    - Uses `useQuery` with queryKey `['features', 'registry']`.
    - queryFn: `api.get('/features/registry')` -> response.data as FeatureRegistryEntry[].
    - staleTime: 5 * 60_000 (5 minutes).
    - Returns the useQuery result (data, isLoading, error, refetch).

    Implement `useSetFeatureFlag()` mutation hook:
    - Uses `useMutation` with mutationFn: `api.post(\`/features/\${slug}/flags\`, { enabled, scope, scope_id })`.
    - onSuccess: invalidate queries `['features', 'resolved']` and `['features', 'registry']`.
    - Returns the useMutation result.

    Implement `useDeleteFeatureFlag()` mutation hook:
    - Uses `useMutation` with mutationFn: `api.delete(\`/features/\${slug}/flags?scope=\${scope}&scope_id=\${scope_id ?? ''}\`)`.
    - onSuccess: same invalidation as set.
    - Returns the useMutation result.

      function FeatureGateRoute({ slug, children }: { slug: string; children: React.ReactNode }) {
        const { isLoading } = useFeatureEnabled(slug)
        if (isLoading) return <>{children}</>  // fail-open: show during load
