  <action>
Create `migration-tool/src/parser.rs` with regex-based SQL dump parser.

The parser must handle the mysqldump 10.13 format observed in `references/app_uoj233.sql`:

1. `parse_dump(content: &str) -> ParsedDump`:
   - Scan each line for `INSERT INTO \`table_name\` VALUES (...)` pattern
   - Extract table name from the INSERT statement using regex: `INSERT INTO \x60(\w+)\x60 VALUES (.+);`
   - Pass the VALUES portion to `parse_values()`

2. `parse_values(values_str: &str) -> Vec<Vec<String>>`:
   - Parse parenthesized groups: `(val1,'val2',...),(val1,'val2',...)`
   - Handle these MySQL value types:
     - String values: `'text'` with escaped quotes `\'` and escaped backslashes `\\`
     - Integer/float values: `123`, `1.5`
     - NULL values: `NULL`
     - Hex-encoded blobs: `0x4142434445...` (preserve as-is, decode later)
     - Empty strings: `''`
   - Strategy: character-by-character state machine that tracks:
     - Inside parentheses (nesting level)
     - Inside single-quoted string (toggle on `'`, but skip `\'`)
     - Current field value being accumulated
   - Do NOT try to use regex to split -- the escaping rules make regex fragile

3. Write comprehensive unit tests in parser.rs using `#[cfg(test)]` module.
   Test with inline SQL strings, not file reads. Cover all 8 behaviors listed above.

4. Add `pub mod parser;` and `pub mod models;` to main.rs module declarations.
