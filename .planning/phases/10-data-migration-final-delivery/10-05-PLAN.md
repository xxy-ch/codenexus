   With transparent MD5 migration logic per D-10-2:
   ```rust
   // AFTER:
   if user.password_hash.starts_with("{MD5}") {
       // Transparent MD5 migration path (D-10-2)
       let md5_hash = &user.password_hash[5..]; // strip "{MD5}" prefix
       let digest = md5::Md5::digest(req.password.as_bytes());
       let computed = format!("{:x}", digest);
       if computed == md5_hash {
           // Upgrade to bcrypt
           let new_hash = bcrypt::hash(&req.password, bcrypt::DEFAULT_COST)?;
           sqlx::query("UPDATE users SET password_hash = $1, updated_at = NOW() WHERE id = $2")
               .bind(&new_hash)
               .bind(user.id)
               .execute(&self.pool)
               .await?;
       } else {
           return Err(anyhow::anyhow!("Invalid credentials"));
       }
   } else {
       // Normal bcrypt verification
       bcrypt::verify(&req.password, &user.password_hash)?
           .then_some(())
           .ok_or_else(|| anyhow::anyhow!("Invalid credentials"))?;
   }
   ```

   Key implementation details:
   - `md5::Md5::digest()` returns a generic array; format with `{:x}` to get hex string
   - MD5 comparison is direct string comparison (the MD5 hash is unsalted, so timing attack is not a concern for legacy hashes that are being upgraded on first use)
   - The bcrypt upgrade happens BEFORE generating tokens, so the user gets a valid session after the transparent upgrade
   - The `updated_at = NOW()` is set explicitly to trigger the updated_at column update
   - No logging of password hashes or plaintext passwords -- only log the username and "MD5 password upgraded to bcrypt" at info level
   - The error message for failed MD5 login is identical to normal failed login ("Invalid credentials") to prevent user enumeration

3. Add `use md5::Digest;` import at the top of service.rs (needed for the `digest()` method on the Md5 hasher). Note: the crate is `md-5` on crates.io but the Rust import name is `md5`.

   /// Returns Ok(true) if matched (caller should upgrade), Ok(false) if not matched.
   /// Returns Err if the hash format is invalid.
   fn verify_md5_password(password: &str, md5_hash: &str) -> bool {
       let digest = md5::Md5::digest(password.as_bytes());
       let computed = format!("{:x}", digest);
       computed == md5_hash
   }
