# Plan 02: Create domain-submissions Crate

<objective>
Extract the submissions module from the api crate into an independent `domain-submissions` workspace crate. The key challenge is the `queue.rs` file which imports `crate::redis` for `create_stream` and `add_message` helpers. Per research recommendation, these ~15-line helpers are duplicated directly into `domain-submissions/queue.rs` rather than adding the full `redis` crate as an api-infra dependency.

Purpose: Submissions has no cross-domain dependencies -- only a redis dependency resolved by inlining helpers. Clean extraction.

Output: Independent `domain-submissions` crate that compiles with `cargo build -p domain-submissions`.
</objective>
