## Tab Content Contracts

### Tab 1: Problem Import

**States:** idle -> uploading -> previewing -> executing -> completed

| State | UI Elements |
|-------|-------------|
| idle | File upload drop zone (dashed border, center-aligned), accepts .zip only. "Select ZIP file" button (outline variant) + drag hint text. Below: expected format description card (collapsed, expandable). |
| uploading | Skeleton placeholders for preview table. Loading spinner. |
| previewing | Summary bar: N total / M valid / S skipped / E errors (uses Badge component with variant colors). Preview table with columns: Title, Difficulty, Test Cases, Status (valid/duplicate/error). Each row status uses Badge: default for valid, secondary for duplicate, destructive for error. "Confirm Import" button (primary, disabled if valid=0). "Cancel" link button. |
| executing | Progress state: indeterminate progress bar (accent color). "Importing..." text with animated spinner. Confirm button shows disabled + loading state. |
| completed | Result summary: N created / S skipped / E errors. Collapsible lists for each category. "Download Report" button (outline) for structured JSON log. "Import Another" button (outline) resets to idle. |

**Upload zone specs:**
- Min height: 160px
- Border: 2px dashed `--border`
- On drag-over: border transitions to solid `--primary`, background `--primary/5`
- Accepted: .zip only (`accept=".zip"`)
- Max file size: 50MB (client-side check before upload)

**Preview table specs:**
- Use existing `table.tsx` component pattern
- Columns: Title (text), Difficulty (Badge with difficulty-* color), Test Cases (count), Status (Badge)
- Max visible rows: 10 with scroll (max-h-80 overflow-y-auto)
- Row count summary above table

### Tab 2: Problem Export

**States:** idle -> exporting

| State | UI Elements |
|-------|-------------|
| idle | Problem selector: search input + multi-select table of problems (title, difficulty, test case count). "Select All" / "Deselect All" buttons. Selected count badge. "Export ZIP" button (primary, disabled when selection empty). |
| exporting | Button shows loading spinner. File download triggers via blob response. Toast notification on completion: "Exported N problems successfully". |

**Problem selector specs:**
- Search input at top (reuse Input component)
- Table with checkbox column (leftmost)
- Max display: 20 rows with pagination (matches existing admin table pattern)
- Selected count shown as Badge next to Export button

### Tab 3: User Import

**States:** idle -> uploading -> previewing -> executing -> completed

| State | UI Elements |
|-------|-------------|
| idle | Two fields: (1) File upload drop zone for CSV, accepts .csv only. (2) Default password input field (required, min 6 chars). Below: expected CSV format description card showing header row and example. "Validate CSV" button (primary, disabled until file selected and password filled). |
| uploading | Skeleton placeholders for preview table. |
| previewing | Summary bar: N total / M valid / S skipped / E errors. Preview table with columns: Username, Role, Campus, Display Name, Email, Status. Same Badge variants as problem import. "Confirm Import" button (primary, disabled if valid=0). |
| executing | Same progress state as problem import. |
| completed | Same result summary pattern. Additional: "skipped" items show duplicate username reason. "Import Another" resets form including password field. |

**Default password field:**
- Input component with type="password"
- Label: "Default Password" (font-medium, text-sm)
- Helper text: "All imported users will receive this password. They should change it on first login."
- Validation: required, min 6 characters
- Show/hide toggle (eye icon button)

**CSV format description:**
- Code block with monospace font (Fira Code)
- Shows header row: `username,role,campus_id,display_name,email`
- Shows 2 example rows
- Copy button in top-right corner

### Tab 4: User Export

**States:** idle -> exporting

| State | UI Elements |
|-------|-------------|

## Interaction Patterns

### File Upload Pattern
1. User clicks upload zone or drags file onto it
2. Client validates file extension (.zip or .csv) and size (50MB ZIP, 10MB CSV)
3. If invalid: inline error below upload zone, no API call
4. If valid: POST to `/imex/import/{type}/validate` as multipart/form-data
5. Show skeleton loading during validation
6. On response: render preview table with status badges

### Preview-to-Confirm Pattern (Two-Phase Import)
1. Validation endpoint returns `{ token, total, valid, warnings, errors, preview_items }`
2. Preview table rendered with per-item status
3. User clicks "Confirm Import" -> Dialog confirmation shown
4. User confirms -> POST to `/imex/import/{type}/execute` with `{ token }`
5. Button enters loading state (disabled + spinner)
6. On success: show result summary, hide preview
7. On token-expired error: show expired message, reset to idle
8. On other error: toast notification with error message, keep preview visible

### Export Download Pattern
1. User selects items or applies filters
2. Clicks "Export ZIP" / "Export CSV"
3. API called with `responseType: 'blob'`
4. Button shows loading spinner during download
5. On response: trigger browser file save via `URL.createObjectURL(blob)`
6. Toast notification confirms success

### Tab State Isolation
- Each tab maintains its own state independently
- Switching tabs does not lose upload progress or preview data
- Each tab has its own query/mutation via TanStack React Query

## Accessibility Notes

- Upload drop zone: keyboard accessible via hidden file input + visible button, aria-label on drop zone
- Preview tables: use `<th scope="col">` headers, proper `<table>` semantics
- Confirmation dialog: focus trap, Escape to close, aria-describedby for body text
- Status badges: include aria-label with full status text (not just color)
- Tab navigation: existing Tabs component uses proper ARIA tablist/tab/tabpanel pattern
- Loading states: aria-live="polite" on result regions
