# Plan 01: Core Role Enum + RBAC + Database Migration

**Goal:** Replace `OrganizationAdmin` with `GradeAdmin` in the shared Role enum, update RBAC permission mapping, and create a database migration. This is the foundation that all other plans depend on.

## Context

The Role enum in `shared/src/models/role.rs` currently has `OrganizationAdmin` as the second-highest role. Phase 13 restructures to:

```
Root > CampusAdmin > GradeAdmin > Teacher > TeachingAssistant > Student
```

Key: GradeAdmin sits **below** CampusAdmin (grade-level admin within a campus), whereas OrganizationAdmin was **above** CampusAdmin (org-wide admin). This reordering affects hierarchy comparisons.

## Tasks

### T-01: Update Role Enum

**File:** `backend/shared/src/models/role.rs`

1. Rename variant `OrganizationAdmin` → `GradeAdmin`
2. Update doc comments: "Grade-level administrator managing classes within specific grade(s)"
3. Update `as_str()` mapping: `Role::GradeAdmin => "gradeadmin"`
4. Update `FromStr`: `"gradeadmin" | "organizationadmin" => Ok(Role::GradeAdmin)` — keep old string as alias for backward compat during migration
5. Reorder hierarchy array to: `[Student, TeachingAssistant, Teacher, GradeAdmin, CampusAdmin, Root]`
6. Update all test references: `Role::OrganizationAdmin` → `Role::GradeAdmin`, `"organizationadmin"` assertions → `"gradeadmin"` or keep as alias test
7. Update serialization test: `assert_eq!(serialized, "\"gradeAdmin\"")`

### T-02: Update RBAC Permission Mapping

**File:** `backend/api-infra/src/rbac.rs`

1. Rename `Role::OrganizationAdmin =>` arm to `Role::GradeAdmin =>`
2. Update GradeAdmin permissions (grade-scoped, NOT org-scoped):
   - KEEP: ManageUsers, ViewUsers (within grade scope)
   - KEEP: ManageProblems, ViewAllProblems
   - KEEP: ManageContests (within grade)
   - KEEP: ManageClasses, ManageAssignments (core GradeAdmin responsibility)
   - KEEP: ViewLeaderboard, ViewStatistics, ViewClassStats
   - REMOVE: ManageOrganization (GradeAdmin does NOT manage organizations)
   - ADD: GradeSubmissions (GradeAdmin grades within their grade)
3. Verify CampusAdmin still has ManageCampus (not affected)

### T-03: Create Database Migration

**File:** NEW `backend/api/migrations/026_reorg_to_gradeadmin.sql`

```sql
-- Phase 13: Replace organizationadmin role with gradeadmin
-- Data migration: existing orgadmins become gradeadmins
-- CHECK constraint update for new role name

BEGIN;

-- Migrate existing data
UPDATE user_roles SET role = 'gradeadmin' WHERE role = 'organizationadmin';

-- Update CHECK constraint
ALTER TABLE user_roles DROP CONSTRAINT IF EXISTS user_roles_role_check;
ALTER TABLE user_roles ADD CONSTRAINT user_roles_role_check
  CHECK (role IN ('root', 'campusadmin', 'gradeadmin', 'teacher', 'teachingassistant', 'student'));

COMMIT;
```
