# Plan 04: Frontend Role Changes

**Goal:** Update the frontend Role type, role arrays, role labels, and UI components to use `gradeAdmin` instead of `organizationAdmin`.

## Context

The frontend defines the Role type in `types/auth.ts`. The backend serializes the new GradeAdmin variant as `"gradeAdmin"` (camelCase via serde `rename_all`). Frontend must match this exact string. Components that use `isAdmin()` and `isTeacherOrAbove()` automatically pick up changes to the `ADMIN_ROLES` and `TEACHER_ROLES` arrays.

## Tasks

### T-01: Update Role Type and Arrays

**File:** `frontend/src/types/auth.ts`

1. **Role type (line 4):**
   ```typescript
   // Before
   | 'organizationAdmin'
   // After
   | 'gradeAdmin'
   ```

2. **ADMIN_ROLES (line 11):**
   ```typescript
   // Before
   export const ADMIN_ROLES: readonly Role[] = ['root', 'organizationAdmin', 'campusAdmin']
   // After
   export const ADMIN_ROLES: readonly Role[] = ['root', 'campusAdmin', 'gradeAdmin']
   ```

3. **TEACHER_ROLES (line 14):**
   ```typescript
   // Before
   export const TEACHER_ROLES: readonly Role[] = ['root', 'organizationAdmin', 'campusAdmin', 'teacher']
   // After
   export const TEACHER_ROLES: readonly Role[] = ['root', 'campusAdmin', 'gradeAdmin', 'teacher']
   ```

4. **roleLabel function (line 28):**
   ```typescript
   // Before
   case 'organizationAdmin': return '机构管理员'
   // After
   case 'gradeAdmin': return '年级管理员'
   ```

5. **File comment (line 1):** Update to match new canonical path.

### T-02: Update UserManagement Role Badge

**File:** `frontend/src/pages/admin/UserManagement.tsx`

1. **Line 97** — Role badge configuration:
   ```typescript
   // Before
   organizationAdmin: { label: '机构管理员', color: 'bg-rose-100 text-rose-700 dark:bg-rose-900/30 dark:text-rose-300' },
   // After
   ```

   Note: Changed color from rose to amber to visually distinguish GradeAdmin from CampusAdmin (which should remain distinct).
