# 交付演示脚本（2026-03-06）

## 1. 演示准备

- 环境：预发布环境（与生产配置一致）。
- 账号：
  - 普通用户账号 1 个
  - 管理员账号 1 个
- 数据：至少 1 个可提交题目、1 个竞赛、1 条讨论、1 篇博客。

## 2. 10 分钟演示流程

1. 登录与首页（1 分钟）
   - 使用普通用户登录。
   - 展示 Dashboard 指标卡与近期活动。

2. 做题与提交（3 分钟）
   - 进入题目列表 -> 打开题目详情 -> 进入 IDE 提交代码。
   - 打开提交历史，展示状态流转和详情页面。

3. 社区功能（2 分钟）
   - 进入讨论区：创建回复并点赞。
   - 进入博客区：打开文章详情、评论、点赞、进入编辑页。

4. 竞赛与榜单（1 分钟）
   - 展示竞赛详情。
   - 跳转榜单页面，展示排名表格。

5. 教师/管理功能（2 分钟）
   - 教师：班级管理页面（列表 + 学生统计列）。
   - 管理员：测试用例配置页（新增或删除一条用例）。

6. 发布与风险（1 分钟）
   - 展示契约清单与发布回滚 runbook。
   - 明确阻塞项处置决策（私信、反作弊）。

## 3. 失败兜底话术

- 若私信/反作弊后端未就绪：
  - 明确说明“前端入口已完成，后端接口待补齐，当前发布策略为隐藏入口/灰度”。

- 若单个接口偶发失败：
  - 切换到已准备的静态数据页或录屏，继续演示主链路不受阻部分。

## 4. 演示成功标准

- 主链路（登录 -> 做题 -> 提交 -> 查看结果）完整跑通。
- 社区和管理最少各 1 条交互成功。
- 阻塞项与发布决策透明、可执行。
