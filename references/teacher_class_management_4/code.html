I've been trying to solve this using the standard two-pointer approach, but I'm getting a TLE on test case 45. Is there a specific edge case with very long lists that I'm missing?

Here is my snippet:

def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return slow # Incorrect return?
    return None

Any help would be appreciated on how to find the entry point of the cycle efficiently.

The issue is that when slow == fast, you are meeting inside the cycle, but that is not necessarily the start of the cycle. To find the entry point, you need a second phase.

Algorithm:

  1. Detect cycle using slow/fast pointers.
  2. If they meet, reset one pointer to head.
  3. Move both pointers one step at a time until they meet again. That meeting point is the cycle entry.
if slow == fast:
    slow = head
    while slow != fast:
        slow = slow.next
        fast = fast.next
    return slow
